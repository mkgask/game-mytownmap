# 貢献ガイド — ドキュメントワークフロー（簡易）

このリポジトリでは、AIエージェント関連の開発ドキュメントに関して次のルールに従います。

主なポイント:

- AIエージェントの思考や設計メモなどの開発ドキュメントは**英語で作成**してください。日本語は英語の翻訳として配置してください。
- 開発ドキュメントは `.docs-ai-agent` に `*.md` として作成します。（speckit のフィーチャードキュメントは `.docs-ai-agent/spaces/[###-feature-name]/` に配置してください。）
- 英語ドキュメントの目安は1ファイル約300行です。250行を超えたら分割を検討してください。
- 日本語訳は `.docs-human-ja` に英語のパスを模倣して配置し、ファイル名の末尾に `.ja.md` を付けてください（例: `.docs-ai-agent/guides/foo.md` → `.docs-human-ja/.docs-ai-agent/guides/foo.ja.md`）。
- **必須:** ドキュメントを更新する場合、英語版と日本語版の両方を必ず更新し、同等の意味・意図を保ってください。大きな変更や解釈が必要な変更は、PRの説明に翻訳差分と確認者（訳者）を明記してください。

PR チェックリスト（PRをレビューへ送る前に記入してください）:

- [ ] 英語版と日本語版の両方を同等の意味で更新しました（更新ファイルをPR本文に列挙してください）
- [ ] `python3 scripts/check_docs_sync.py` を実行し、報告された問題を修正しました
- [ ] 翻訳の検証者（必要に応じて）をPRに記載しました

スクリプトとチェック:

- ドキュメントの整合性と行数を検査するスクリプトは `scripts/check_docs_sync.py` にあります。
- ドキュメントやスクリプトに対する push / PR では GitHub Actions でドキュメントチェックが走る場合があります。

ローカルでの実行方法:

```bash
python3 scripts/check_docs_sync.py
```

チェッカーが非ゼロで終了した場合は、報告された問題を修正してからマージしてください。

ドキュメントを正確に保ち、翻訳も整備してくれてありがとうございます！ 🎯
