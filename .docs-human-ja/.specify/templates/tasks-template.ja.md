# Tasks: [FEATURE NAME]

**入力**: `specs/[###-feature-name]/` の設計ドキュメント
**前提**: plan.md（必須）、spec.md（ユーザーストーリー必須）、research.md、data-model.md、contracts/

**テスト**: 以下の例はテストタスクを含みます。テストが明示的に要求された場合のみ含めてください。

**スコープガード**: プレイヤー向けの作業は道路/建物の追加・変更・削除に限定。その他のインタラクションはガバナンス承認が必要。

**ドキュメント更新ルール**: 開発ドキュメントを更新する際は、必ずリポジトリルートの `CONTRIBUTING.md` を参照し、その指示に従ってください。PRに参照を記載してください。

**UI/E2E テスト**: ブラウザ向けテストは Playwright を使用すること。他の UI/E2E フレームワークを使う場合はガバナンス承認が必要です。

**編成方針**: タスクはユーザーストーリー単位で独立実装・独立テストができるようにまとめます。

## フォーマット: `[ID] [P?] [Story] Description`

- **[P]**: 並列実行可（別ファイル・依存なし）
- **[Story]**: 紐づくユーザーストーリー (例: US1, US2)
- ファイルパスを明記してください

## パス規約

- **シングルプロジェクト**: `src/`, `tests/` をリポジトリルートに配置
- **Webアプリ**: `backend/src/`, `frontend/src/`
- **モバイル**: `api/src/`, `ios/src/` または `android/src/`
- 以下の例はシングルプロジェクト想定。plan.md に合わせて修正してください。
