# 画面機能仕様書: タイトル画面

## 目的
- 新規ゲーム開始、最近のセーブ再開、その他メニュー（チュートリアル/コンフィグ/パッケージライセンス）への導線を提供する。
- セーブ件数が多い場合は専用のデータ選択画面へ遷移し、タイトル画面の情報量を抑える。

## 画面レイアウト
- 上段: ゲームタイトルロゴを中央配置。
- 下段: 左右2カラム構成。
  - 左カラム: 新規ゲームボタン + 最近のセーブリスト（最新3～5件）。
  - 右カラム: ゲームメニュー（チュートリアル、コンフィグ、パッケージライセンス）。

## 要素別仕様
### 共通
- 入力手段: クリック/タップ/キーボード(Enter/矢印/Tab)対応を想定。
- フォーカス管理: メニュー項目間を矢印キーまたはTabで移動できること。

### 新規ゲームボタン
- 文言: 「新規ゲーム」
- 挙動: 押下で新規セッションを開始し、ゲーム開始フローへ遷移。
- 前提: 新規開始時はデフォルト設定（経路探索難易度はコンフィグの現在値）を適用。

### 最近のセーブリスト
- 件数: 最新3～5件を表示。5件を上限とし、セーブ数がそれ以下なら全件表示。
- 並び順: 更新日時の新しい順。
- 表示項目: セーブ名/スロットID、最終更新日時、進行概要（任意で日数やスコア）。
- 挙動: 項目押下で該当セーブをロードして再開。
- 5件を超える場合: 「もっと見る」(または「すべて表示」) を表示し、データ選択画面へ遷移。

### データ選択画面（遷移先の要求）
- 役割: 全セーブを一覧/検索/スクロールで選択できること。
- 帰還動線: タイトル画面へ戻るボタンを必須。

### ゲームメニュー（右カラム）
- チュートリアル
  - 状態: ゲームがある程度動作した段階で実装予定。現時点ではプレースホルダー/無効化（ラベルに「準備中」等を明示）でも可。
  - 目的: 基本操作と目標（渋滞緩和、道路/建物編集、シード再現性）を案内。
- コンフィグ
  - 最低限の項目: 経路探索難易度モード選択（Easy/Normal/Hard）。
  - 将来拡張: 音量や表示設定を追加予定でプレースホルダー枠を確保。
- パッケージライセンス
  - 内容: ビルド時点で使用中のパッケージ名とライセンスを列挙した静的ページへの遷移。
  - 生成: ビルド工程で自動生成し、タイトル画面メニューから参照可能にする。

## 状態・遷移
- ロゴ表示 → ボタン/メニュー初期化 → セーブリスト取得 → 表示完了。
- 「新規ゲーム」押下: 新規セッション初期化後、ゲーム本編へ遷移。
- 最近のセーブ押下: 対応するセーブをロード → ゲーム本編へ遷移。
- 「もっと見る」押下: データ選択画面へ遷移。
- メニュー各項目押下: 対応画面へ遷移（チュートリアル未実装時はプレースホルダー挙動）。

## エラー/例外ケース
- セーブ読み込み失敗: エラーメッセージを表示し、再試行/別データ選択/タイトルへ戻るの選択肢を提示。
- セーブが0件: 最近のセーブリストは非表示または「セーブがありません」表示のみ。新規ゲームボタンは有効。
- ビルド済みライセンスリストが欠落: メニュー項目は非活性化し、New Relicに警告ログを送信する。ゲーム本筋と無関係なためトーストやダイアログは表示しない。

## アクセシビリティ
- キーボード操作: 主要要素にフォーカスリング。Enter/Spaceでアクション、Escでキャンセル/戻る。
- スクリーンリーダー: ロゴに代替テキスト。ボタンとリスト項目に役割/ラベルを付与。

## テレメトリ (方針レベル)
- タイトル表示、各ボタン押下、セーブロード結果（成功/失敗）をイベントとして計測可能にする。ただしPIIやセーブ内容の詳細は送信しない。
