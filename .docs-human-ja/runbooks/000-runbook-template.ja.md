# 000 RunBook テンプレート

**目的:** この RunBook が実行するタスクと期待される結果を説明します。

**RunBook メタデータ**
- RunBook 番号: `000`（作成時に 3 桁番号に置き換えてください）
- 短いタイトル: `<short-hyphenated-title>`（ファイル名に使用）
- 重大度 / 影響: `low|medium|high`
- タグ: `例: deploy, db, emergency`

**前提条件**
- アクセス、資格情報、環境（例: ブランチ、クラスター、リージョン）など。
- 想定されるダウンタイムやリスクの注意点。

**影響範囲**
- この作業によって影響を受ける可能性のあるすべてのシステム、サービス、リージョン、ユーザー、データ、設定を列挙してください。
- 各項目について想定される影響のレベル（読み取りのみ / 書き込み / ダウンタイム）と派生的な影響を記載してください。
- リスク評価に含めるべき接続点や依存関係を明示してください。

**テスト範囲**
- 影響範囲に対してその接続点の“一つ外側”までを含めたテストを定義してください（変更が入っていないはずの近接するシステムが正しく動作していることを確認すること）。
- 各テストについて具体的な検証手順、期待値、失敗判定基準を列挙してください。
- スモークテストや監視チェックを含め、境界での回帰検出方法を明示してください。

**手順**
1. 準備
   - 前提条件の確認: 実行するコマンド、確認すべきアーティファクト。
2. 実行
   - 手順ごとのコマンド（正確に記述してください。`set -e` などのシェルチェックを含めても良いです）。
   - 期待される出力と失敗の検出方法を記載。
3. 検証
   - 成功の検証方法（ヘルスチェック、スモークテスト、メトリクス、UI）。

**ロールバック / 対処**
- 失敗した場合に実行する手順。
- 所要時間の見積もりやオンコールへの連絡方法。

**後処理 / クリーンアップ**
- クリーンアップの手順、監視ウィンドウ、フォローアップ。

**連絡先 & エスカレーション**
- プライマリ: `@primary`
- セカンダリ / オンコール: `@on-call`

**参照**
- 関連ドキュメント、PR、RunBook、監視ダッシュボード、チケットへのリンク。

# 利用方法
- RunBook は `.docs-ai-agent/runbooks/` の該当する 100 番区切りディレクトリ（例: `1-100/`, `101-200/`）に配置してください。
- ファイル名は `<NNN>-<short-hyphenated-title>.md`（例: `001-deploy-website.md`）としてください。
- `000-runbook-template.md` をコピーして RunBook を作成し、番号とタイトルを更新してください。
- 同じ相対パスに対応する日本語訳を `.docs-human-ja/runbooks/...` に `.ja.md` として追加してください。
